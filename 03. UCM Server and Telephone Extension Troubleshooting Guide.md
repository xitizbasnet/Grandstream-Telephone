**System:** Grandstream UCM Server

**Department:** IT TEAM

**Domain:** INFRA & SECURITY

---
  

# 1 🎯 Purpose

This document provides a step-by-step procedure for diagnosing and resolving telephone-related issues by configuring the UCM Server and the corresponding IP Telephone Extension. It ensures consistent handling of VoIP extension issues within the IT Department.

---

# 2 📌 Scope

This guide applies to all IT personnel responsible for managing and maintaining UCM (Unified Communications Management) Servers and IP Telephone Devices within the organization’s telephony network.

---

# 3 ⚙️ Prerequisites

Before performing the configuration steps, ensure the following:

• Administrative credentials for both the UCM server and the IP phone.

• Network access to both devices.

• Stable power and network connectivity.

---

4. Click Login
5. Once logged in, the UCM Server Dashboard will appear. The navigation panel on the left contains all configuration options.

---

# 5 ⚙️ Extension Configuration

1. From the left-hand navigation panel, select “Extension/Trunk”.

2. Click on the sub menu “Extensions”

3. Locate the search Icon for the Extension.

4. Type your telephone Extension here.

   o e.g., Extension 912

6. Click Edit to open the configuration dashboard for that extension.

7. Once clicking on the Edit, the Edit configuration dashboard will appear.

8. Navigate to the “Media” tab.

9. Navigate to the “Codec Preferences” tab to review.

10. Ensure that the following codecs are selected for optimal performance:

    
   • PCMU
   
   • PCMA

11. Click Save to store the updated configuration.

Note: Proper codec selection ensures smooth audio transmission and compatibility between the server and client devices.
Once saved, the server-side configuration is complete.

---

# 6 📞 Client (Telephone) Configuration

Next, configure the IP Telephone extension.

1. From the UCM Server dashboard, identify the IP Address assigned to the extension.

    o For example, Extension 912 corresponds to IP Address: 192.168.1.100.
   
3. Open your web browser and enter the IP address (without the port number):

   • URL: [http://192.168.1.100/login](http://192.168.1.100/login)
   
4. Log in using the following credentials:

 • Username: Admin

 • Password: ********

4. Click Login
5. The IP Telephone Dashboard will appear. Use the navigation bar on the left to access settings.
6. Navigate to Accounts
7. Click Accounts.
8. Open the Codec Settings section.
9. Under the “Preferred vocoder”, select the following options:
    
   • PCMU
   
   • PCMA
   
10. Click Save to apply the configuration.

At this stage, both the server and client configurations are properly synchronized.

---

# 7 🔄 Reboot and Testing Procedure

1. Reboot the client (telephone) device to apply the configuration changes.

    o Locate the Power/Reboot icon on the top-right corner of the navigation bar.
   
3. Select OK, when confirm prompted.
4. Wait for the device to restart completely.
5. Once rebooted, test the telephone to verify that the extension is functioning correctly.

---

# 8 🛠 Troubleshooting and Escalation

If the issue persists after following the above configuration steps:

1. Confirm network connectivity between the UCM Server and the IP phone.
2. Recheck the codec configuration on both devices.
3. Reboot both the server and the client device if necessary.
4. If the problem remains unresolved:
   
   o Escalate to the IT Team Lead for further analysis.
   
   o If still unresolved, coordinate with the vendor for advanced troubleshooting and support.
   

---

# 9 📘 Conclusion

In certain cases, telephone extension issues occur due to misconfigured codec preferences or outdated device settings.

Following the above configuration steps and reboot procedure typically resolves the issue. If the issue continues, escalate internally to the IT department and coordinate externally with the OMNETWORK Vendor for advanced resolution.
